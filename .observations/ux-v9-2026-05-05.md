## Walk Script — v9 Detector Improvements

v9 over v8 on the director check:
- v8 used `el.innerText === "Director"` to find the card, then climbed to parent. The UUID lived in a sibling text node — parent climb missed it, causing `DIRECTOR_OCCUPANT_RESOLVED` false positive while body text showed the UUID.
- v9 uses raw `bodyTextSample` substring scan: check if the known user UUID `bbbd909d-02ab-4ea6-9da2-98d10d4aeba8` appears within 200 chars of "Director". Ground truth is body text, not DOM element traversal.
- Result: v9 correctly reports `DIRECTOR_NAME_RESOLVED` because UUID is absent from body text.
