## New Issues (v6 discoveries)

### N1: `AEQI_UPPERCASE_STRUCTURAL` detector over-fires on session-rail ASIDE
The structural AEQI check queries `nav, header, ASIDE, [class*=sidebar]` which captures the session rail on agent-overview/sessions. That rail contains user-typed session titles including "improve aeqi tiself" (lowercase — but the structural detector fires 7x because the selector scope spans the full rail). The `AEQI_UPPERCASE_TOTAL` shows only 1x, confirming it's user content. Needs detector fix: exclude ASIDE/session-sidebar from the structural scan, or switch to walking only non-content-slot ancestors. Wave 20 detector refinement.

### N2: `AEQI_UPPERCASE_STRUCTURAL` in docs nav — "AEQI Entity & AA"
Docs sidebar nav section heading. Lives in aeqi-docs repo, not apps/ui. Needs a docs-side rename: "aeqi Entity & AA" → comply with brand lowercase rule. Low-priority standalone fix.
