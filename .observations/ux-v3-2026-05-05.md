## System-wide findings

### SW-1 — Pill buttons everywhere (P1, unaddressed from v1)

Every authed route has 5–19 pill buttons detected. Sidebar circular icon controls (50px radius) appear on every page. Action buttons ("Save", "Change", "Continue", "Email me a code") have 999px radius.

Sidebar icon controls (50px): These are circular avatar/icon buttons — circular shape may be intentional for avatar rendering. Needs design decision: if circular avatar buttons are intentional, exclude them from the button-radius rule. If not, apply `border-radius: var(--radius-sm)`.

Action buttons (999px): These are definitely wrong. "Continue" on signup, "Email me a code" on signin, "Save" on settings — all pill-shaped. One CSS change to `.btn-primary, .btn-secondary` with correct radius token fixes ~20+ surfaces.

Cookie consent buttons on landing: both pills. Same fix.

### SW-2 — Hairlines system-wide (P2, unaddressed from v1)

6–14 elements with 1px borders detected on: signin (6), app-root (8), me-root (10), me-agents (13), me-quests (9), me-ideas (9), me-events (8), me-settings (6), company-roles (9), agent-quests (9), agent-events (7), agent-ideas (9), blueprints (14). Per `feedback_no_hairlines.md`: "No 1px borders for dividers, separators, or card outlines. Use spacing, tint shifts, and weight to express boundaries." These hairlines are violating the locked design rule.

### SW-3 — Personal rail leaks company-only tabs (P1, noted in v2 as N-1)

All /me/* routes show ORGANIZATION: Treasury · Ownership · Governance · Roles in the sidebar. Per `project_personal_rail_v1.md`: personal entity rail hides Roles/Ownership/Governance (degenerate). Correct personal rail: Inbox · Agents · Events · Quests · Ideas · Treasury · Settings. The ORGANIZATION section should not appear on personal entity pages.

### SW-4 — "AEQI" uppercase in agent system identity leaks everywhere (P1)

The root cause is the agent's system identity text containing "AEQI" uppercase: `You are an AEQI agent named "Luca Eich"`. This text:
- Renders directly in the company/personal overview mission field
- Seeds all agent-generated ideas and quests with "AEQI" uppercase
- Results in 345 ideas containing "AEQI" uppercase across the ideas and agent-ideas views

Fix: lowercase "aeqi" in all default agent identity templates, system prompts, and event injection text. This is a two-part fix: (a) rename "AEQI" → "aeqi" in identity text; (b) decide whether to show the raw identity text in the mission field or replace with an empty state + CTA.
