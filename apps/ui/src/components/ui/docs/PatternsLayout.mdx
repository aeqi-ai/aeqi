# Layout

`<Page>` and its companion primitives are the canonical starting point
for app screens. `<Stack>` and `<Inline>` are still the right local
composition tools inside a card, form, or small component body. Together
they replace the `style={{ display: 'flex', gap: ... }}` and one-off
page wrapper patterns scattered across the codebase.

Use Page primitives for page measure, header actions, toolbar zones,
section rhythm, and metric summaries. Use Stack and Inline for local
vertical or horizontal grouping after the page shell already exists.

## When to use

- **Page layouts** — `Page` + `PageHeader` + `PageBody` for screens.
- **Page sections** — `PageSection` for section titles, descriptions,
  actions, and body rhythm.
- **Toolbar rows** — `PageToolbar` for search/filter/action chrome.
- **Metrics** — `MetricGrid` + `MetricCard` for stat summaries.
- **Card internals** — header, body, and footer as a `<Stack gap="4">`.
- **Form sections** — label + input pairs stacked with `<Stack gap="1">`,
  the whole form with `<Stack gap="4">`.

## When not to use

- **Inside a primitive's own CSS Module.** A component that is
  fundamentally a row — `Tabs`, `IconButton`, `Badge` — already owns its
  flex layout in its `.module.css`. Reaching for `<Inline>` inside that
  component is a layering violation: the primitive should be the source of
  truth for its own geometry.
- **One-off micro-spacing inside a component you're already styling.**
  If you're in a `DetailField.module.css` and need two items 4 px apart,
  use `var(--space-1)` directly in that file. Don't import a layout
  primitive to fix a single interior gap.

## Examples

### 1 — Internal app page

Start new dashboard screens with Page primitives. This gives future
agents the right page measure, header rhythm, toolbar grouping, section
spacing, and metric grid without custom wrapper CSS.

<Page width="wide">
  <PageHeader
    title="Operations"
    description="Runtime work across active company agents, grouped for scan-first review."
    meta={<Badge variant="accent">Company Scope</Badge>}
    actions={
      <>
        <Button variant="secondary" size="sm">
          Export
        </Button>
        <Button variant="primary" size="sm">
          New Quest
        </Button>
      </>
    }
  />
  <PageBody>
    <PageToolbar
      aria-label="Operations controls"
      grow
      actions={
        <Button variant="primary" size="sm">
          Assign
        </Button>
      }
    >
      <Input aria-label="Search quests" placeholder="Search quests" size="sm" />
    </PageToolbar>
    <MetricGrid columns={3}>
      <MetricCard label="Active quests" value="18" detail="Across six agents" />
      <MetricCard label="Blocked" value="2" detail="Needs operator review" />
      <MetricCard label="Done today" value="7" trend="+3" detail="Since 09:00" />
    </MetricGrid>
    <PageSection
      title="Quest queue"
      description="Sorted by the work most likely to need operator attention."
      actions={
        <Button variant="secondary" size="sm">
          View all
        </Button>
      }
    >
      <Stack gap="2">
        {[
          { quest: "Normalize launch flow", owner: "founder-ops", status: "Working" },
          { quest: "Review billing webhook", owner: "runtime", status: "Blocked" },
          { quest: "Publish operator docs", owner: "docs", status: "Done" },
        ].map(({ quest, owner, status }) => (
          <Inline key={quest} justify="between" align="center" gap="3">
            <Stack gap="0">
              <strong>{quest}</strong>
              <span>{owner}</span>
            </Stack>
            <Badge
              variant={status === "Blocked" ? "warning" : status === "Done" ? "success" : "info"}
              size="sm"
              dot
            >
              {status}
            </Badge>
          </Inline>
        ))}
      </Stack>
    </PageSection>
  </PageBody>
</Page>

### 2 — Section with actions

Use `PageSection` instead of hand-writing section headers with flex,
spacer divs, or margin rules. The primitive owns heading/action alignment
and responsive collapse.

<Page width="default">
  <PageSection
    title="Integrations"
    description="Compact section rhythm for settings pages."
    actions={
      <Button variant="secondary" size="sm">
        Add integration
      </Button>
    }
  >
    <MetricGrid columns={3}>
      <MetricCard label="Connected" value="5" />
      <MetricCard label="Needs review" value="1" />
      <MetricCard label="Disabled" value="0" />
    </MetricGrid>
  </PageSection>
</Page>

### 3 — Local rows inside a section

Once the page shell is in place, use `<Stack>` and `<Inline>` for local
row composition. They are still the right way to avoid ad-hoc flex in
small bodies.

<Page width="default">
  <PageSection title="Agent detail">
    <Stack gap="2">
      {[
        { label: "Parent", value: "orchestrator" },
        { label: "Quests in flight", value: "3" },
        { label: "Last event", value: "2 minutes ago" },
      ].map(({ label, value }) => (
        <Inline key={label} justify="between" align="center" gap="3">
          <span>{label}</span>
          <strong>{value}</strong>
        </Inline>
      ))}
    </Stack>
  </PageSection>
</Page>

## Token mapping

The `gap` prop on Page, PageBody, PageSection, Stack, and Inline is a key
into the spacing scale. Values outside this table are a type error.

| `gap` prop | CSS token   | Computed value |
| ---------- | ----------- | -------------- |
| `"0"`      | `--space-0` | 2px            |
| `"1"`      | `--space-1` | 4px            |
| `"2"`      | `--space-2` | 8px            |
| `"3"`      | `--space-3` | 12px           |
| `"4"`      | `--space-4` | 16px           |
| `"5"`      | `--space-5` | 20px           |
| `"6"`      | `--space-6` | 24px           |
| `"8"`      | `--space-8` | 32px           |

> The scale skips `7` because `--space-7` is not defined in `tokens.css`.
> The 4 px grid jumps from 24 px to 32 px at the large end — use `"6"` for
> section breathing room, `"8"` for page-level separation.
